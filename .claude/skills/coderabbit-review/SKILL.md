# CodeRabbitレビュー対応

## 指摘の取得

```bash
gh pr view {PR番号} --comments
gh api repos/{owner}/{repo}/pulls/{PR番号}/comments
```

## 重要度分類

すべてのCodeRabbit指摘を以下で分類する：

| 重要度 | 対応方針 |
|--------|----------|
| Critical | 必ず修正する |
| Major | 修正する |
| Nitpick | コード品質・バグリスク・セキュリティに関するもの → 修正。スタイルのみ・好みの問題 → 見送り可 |

## 対応後のコメント投稿

**すべてのCodeRabbit指摘に対してPRコメントで返信する。**

対応した場合：
```
修正しました。
Why: {なぜこの修正が必要だったか}
対応内容: {何をどう修正したか}
```

対応しない場合：
```
本指摘は対応を見送ります。
Why: {なぜ対応しないか — 理由を明確に}
```

## ルール

- コメントは日本語で記述する
- `gh api`でPRに返信する
- **重複投稿禁止** — 同じ指摘に二重にコメントしない
- **Whyを必ず書く** — 「修正しました」だけのコメントは禁止
