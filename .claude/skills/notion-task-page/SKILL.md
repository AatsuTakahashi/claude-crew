# Notionタスクページ作成

## プロパティ

- **タスクタイトル**: タスクの内容がわかるタイトル
- **担当者**: 指示を出した人を設定する（実行者ではない）
- **ステータス**: 進行状況
- **プロジェクト**: 対象プロジェクト
- **優先度**: 高 / 中 / 低
- **タグ**: 開発、バックエンド、フロントエンド 等
- **開始日 / 期限**: 日付

## 詳細欄（コンテンツ）

以下の構造で記述する：

```
[タイムスタンプ挿入ボタン]
（ボタンを押すと「## @今」のブロックが挿入される）

## Why
（なぜこのタスクが必要なのか。背景・動機・課題を丁寧に詳細に書く。
Whyは最も重要。後から読んだ人が「なぜやったのか」を完全に理解できるように書く。）

## What
- 変更内容1
- 変更内容2
- ...

## How
1. 具体的な実装手順1
2. 具体的な実装手順2
3. ...
```

## 作業メモ（時系列）

作業中の議論・決定・メモをタイムスタンプ付きで時系列に記録する：

```
## @2026-03-19 15:30
- 仕様変更の決定内容
- 議論のメモ

## @2026-03-19 10:00
- 初回の打ち合わせメモ
```

新しいメモが上に来る（降順）。

## ルール

- **Whyを最重要視する** — Whyが薄いページは作らない。背景を徹底的に書く
- **担当者 = 指示者** — 実行者ではなく、指示を出した人を担当者に設定する
- **重複を作らない** — 既存のページがないか確認してから作成する
