# Git 絶対ルール

## コミット粒度（厳守）

**1タスク1コミット。** 複数の独立した変更を1つのコミットにまとめることは禁止。

- 変更の目的が異なる → 必ず別コミットに分ける
- 「一緒にやったから1コミットでいい」という判断は存在しない
- コミット前に「この変更は本当に1つの目的か？」を確認する

## ブランチ運用

- デフォルトブランチは `main`
- 変更を加える際は、最新の `main` から新しくブランチを切って作業する
- **`main` に直接コミットしない**

## コミットメッセージフォーマット（厳守）

**必ず以下のフォーマットで書くこと。Why / What / Next は全コミットで必須。省略禁止。**

```
<emoji> <type>(<scope>): <subject>

## Why
<なぜ今これが必要か — 背景・動機・課題>

## What
<何を変更したか — 変更内容を簡潔に>

## Next
<次に必要な作業>
```

- `scope` は省略可
- `subject` は日本語で簡潔に
- `## Why` / `## What` / `## Next` の見出しを必ず含めること
- 見出しなしで本文だけ書くのは禁止
- Co-Authored-By 行は不要

### タイプ一覧

| タイプ | 絵文字 | 用途 |
|---|---|---|
| feat | ✨ | 新機能 |
| fix | 🐛 | バグ修正 |
| docs | 📝 | ドキュメント |
| style | 💄 | コードスタイル |
| refactor | ♻️ | リファクタリング |
| perf | ⚡ | パフォーマンス改善 |
| test | ✅ | テストの追加・修正 |
| build | 📦 | ビルド関連 |
| ci | 🎡 | CI設定 |
| chore | 🔧 | その他の雑務 |
| revert | ⏪ | リバート |

## プッシュルール

**明示的に指示されない限り、絶対に `git push` を実行しないこと。**
コミットまでは行ってよいが、リモートへのプッシュはユーザーの明示的な許可が必要。
