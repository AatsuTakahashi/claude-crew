# Git ルール

## ブランチ運用

- デフォルトブランチは `develop`
- 変更を加える際は、最新の `develop` から新しくブランチを切って作業する
- `develop` に直接コミットしない

## コミットメッセージ

Conventional Commits 形式に従うこと。

### フォーマット（厳守）

**必ず以下のフォーマットでコミットメッセージを書くこと。自由記述の body は禁止。**

```
<emoji> <type>(<scope>): <subject>

## Why
<なぜ今これが必要か — 背景・動機・課題を書く>

## What
<何を変更したか — 変更内容を簡潔に書く>

## Next
<次に必要な作業を書く>
```

- `scope` は省略可
- `subject` は日本語で簡潔に書く
- **Why / What / Next は全コミットで必須。省略してはならない。**
- `## Why` / `## What` / `## Next` の見出しを必ず含めること。見出しなしで本文だけ書くのは禁止

### プッシュルール

- **明示的に指示されない限り、絶対に `git push` を実行しないこと。**
- コミットまでは行ってよいが、リモートへのプッシュはユーザーの明示的な許可が必要。

### タイプ一覧

| タイプ | 絵文字 | 用途 |
|---|---|---|
| feat | ✨ | 新機能 |
| fix | 🐛 | バグ修正 |
| docs | 📝 | ドキュメント |
| style | 💄 | コードスタイル（動作に影響しない変更） |
| refactor | ♻️ | リファクタリング |
| perf | ⚡ | パフォーマンス改善 |
| test | ✅ | テストの追加・修正 |
| build | 📦 | ビルド関連 |
| ci | 🎡 | CI設定 |
| chore | 🔧 | その他の雑務 |
| revert | ⏪ | リバート |
